Risk Analysis

먼저 팩트 체크: 실제로 얼마나 노출되는가?

모두의 창업 출품 시 아이디어 노출 범위를 정확히 파악합니다

결론부터: 생각보다 노출이 적습니다

모두의 창업 FAQ에 따르면, "한 줄 아이디어"만 선택적으로 공개 가능하고 상세 아이디어 내용은 항상 비공개입니다. 선발 후에도 핵심/원천 기술을 제외한 발전 과정만 본인 동의하에 공유됩니다.

그러나 완전한 비공개는 아닙니다. 멘토(500+명), 심사위원, 운영기관 담당자, IR 참석 투자자 등 내부 관계자에게는 아이디어가 노출됩니다. 2라운드부터는 Public IR도 진행됩니다.

라운드별 노출 범위

낮음

지원서 제출 단계

한 줄 아이디어를 "비공개"로 설정 가능. 상세 내용은 운영기관 책임멘토만 열람. 노출 범위: 1~2명.

낮음

출전자 선발 (4,000명)

1:1 책임멘토링. 멘토 1명에게 상세 공유. AI 솔루션 활용은 자체적이므로 추가 노출 없음.

중간

1라운드 (500명)

기술멘토 추가 배정. 서면 평가 심사위원에게 사업계획서 공개. 노출 범위: 5~10명 내외.

높음

2라운드~결선 (200명~10명)

Public IR 시작. 기술 전문가, 투자자, 선배 창업자 앞에서 발표. 전국 결선은 TV 방송. 노출 범위: 수백~수만 명.

핵심 리스크 포인트

진짜 위험은 "컨셉 카피"가 아니라 "실행 카피"입니다.

"MLB 판타지리그를 주식에 적용한다"는 컨셉 자체는 아이디어 수준이라 법적 보호가 어렵습니다. 실제로 해외에서 이미 유사 컨셉이 존재합니다. 진짜 보호해야 할 것은:

(1) 브랜드 (상표) — "Stock Heist League"라는 이름과 아이덴티티
(2) 스코어링 알고리즘 (특허) — 주식 지표를 판타지 스탯으로 변환하는 구체적 기술
(3) 소스코드와 UI (저작권) — 실제 구현물
(4) 세부 파라미터 (영업비밀) — 알고리즘의 가중치, 튜닝 값 등

Layer 1

상표권 (Trademark) — 브랜드를 선점하라

한국은 선출원주의: 먼저 출원한 사람이 이긴다

상표 출원 — 가장 먼저, 가장 쉽게

한국은 선출원주의 (先出願主義)입니다. 누가 먼저 사용했느냐가 아니라 누가 먼저 출원했느냐가 중요합니다. "Stock Heist League"를 내일이라도 다른 사람이 먼저 출원하면, 당신이 이 이름을 쓸 수 없게 됩니다.

비용: ~25만원 (DIY) 기간: 10~14개월 우선도: 즉시

보호 가능

  • "Stock Heist League" 브랜드명 (영문)
  • 한글 브랜드명 (예: "주식 하이스트 리그" 등)
  • 로고/엠블럼 (도형상표)
  • 앱 아이콘
  • 제품 출시 전에도 출원 가능 (사용 의사만으로 충분)

보호 불가

  • "주식 판타지 리그"처럼 서비스를 직접 설명하는 명칭 (기술적 상표 거절)
  • 게임 메커닉이나 룰 자체

출원 절차

  • KIPRIS 선행조사 — kipris.or.kr에서 동일/유사 상표 검색 (무료)
  • 특허로 전자출원 — patent.go.kr에서 온라인 출원. 제9류(소프트웨어) + 제41류(게임 서비스) 지정
  • 방식심사 — 서류 형식 검토
  • 출원공고 — 상표공보에 30일간 게재 (이의신청 기간)
  • 실체심사 — 거절 사유 검토
  • 등록결정 — 등록료 납부 후 상표권 발생 (10년간 유효, 무한 갱신 가능)

Layer 2

특허 (Patent) — 핵심 알고리즘을 보호하라

BM(비즈니스 방법) 특허로 스코어링 시스템과 게임 메커닉의 기술적 구현을 보호

임시출원 (가출원) — 20만원으로 12개월의 시간을 산다

2023년 3월 30일 도입된 임시명세서 출원 제도. 어떤 파일 형식(PDF, HWP, PPT, JPG)으로든 아이디어를 제출하면 출원일이 확정됩니다. 정식 명세서는 12개월(보정은 14개월) 이내에 제출하면 됩니다.

비용: 200,000원 준비: 1~2일 우선도: 즉시

특허로 보호 가능한 것 (기술적 구현이 구체적일 때)

  • 스코어링 알고리즘: 실시간 시장 데이터를 판타지 포인트로 변환하는 구체적 연산 방법
  • 드래프트/옥션 메커니즘: 실시간 주식 드래프트의 서버-클라이언트 통신 아키텍처
  • 리스크/리워드 계산 엔진: 판타지 스포츠 메커닉과 금융 지표를 결합한 포트폴리오 스코어링
  • 매치메이킹/리그 관리: 참가자 매칭, 리그 밸런싱 서버 사이드 알고리즘
  • 어뷰징 방지: 조작/부정행위 탐지 기술적 방법

특허로 보호 불가능한 것

  • "주식으로 판타지리그를 한다"는 추상적 아이디어 자체
  • 하드웨어 없이 순수한 비즈니스 방법만
  • 수학 공식 그 자체 (기술적 적용이 아닌 경우)
주의 — 선행 특허 존재: 미국 Fandex Inc.가 "판타지 스포츠 주식 거래소 플랫폼" 관련 US Patent No. 11,436,674를 보유하고 있습니다. 한국 출원 시 이와 차별화된 기술적 구현을 강조해야 합니다 (예: AI 기반 다차원 스코어링, KRX 공공데이터 연동 방식 등).

임시출원 → 정식출원 로드맵

  • 지금 (D-0): 스코어링 알고리즘, 시스템 아키텍처를 PPT/PDF로 정리
  • 임시출원 제출: 특허로(patent.go.kr)에서 200,000원에 출원. 출원일 확보
  • 12개월 이내: 변리사와 함께 정식 명세서(청구항 포함) 작성하여 제출
  • 우선심사 청구: 6개월 내 심사 시작 가능 (일반은 12~18개월)
  • 등록 결정: 총 18~24개월 (우선심사 시 12~15개월)

Layer 3

저작권 (Copyright) — 만들면 자동으로 발생

한국은 무방식주의: 창작 즉시 저작권 발생. 등록은 증거력 강화용

자동 보호 + 등록으로 증거력 확보

소스코드, UI 디자인, 그래픽, 텍스트 등 창작물은 만드는 순간 저작권이 발생합니다. 별도 등록 없이도 보호받지만, 한국저작권위원회(CROS)에 등록하면 창작일 추정력과 제3자 대항력을 확보합니다.

비용: 53,600원 기간: 1~2주 우선도: MVP 완성 후

자동 보호되는 것

  • 소스코드 전체
  • UI/UX 디자인, 그래픽 에셋, 아이콘, 일러스트
  • 사운드, 음악, 효과음
  • 캐릭터 디자인, 게임 세계관 설정
  • 튜토리얼/설명 텍스트
  • 데이터베이스 구조 (편집저작물)
  • 게임 룰의 "선택과 배열의 유기적 조합"이 창작성을 가질 때 (대법원 판례)

보호되지 않는 것

  • "드래프트로 주식을 고른다"는 추상적 아이디어
  • 일반적/범용적 UI 패턴
  • 순수 기능적 요소

Layer 4

영업비밀 (Trade Secret) — 공개하지 않는 것이 최선의 보호

특허로 시스템 아키텍처를 보호하되, 세부 파라미터는 영업비밀로 유지

특허 + 영업비밀 = 이중 보호 전략

특허는 공개 대가로 독점권을 얻지만, 영업비밀은 비공개인 한 영구히 보호됩니다. 시스템 아키텍처(특허) + 세부 튜닝 값(영업비밀)로 이중 보호하는 것이 최적 전략입니다.

비용: 무료~최소 우선도: 즉시

영업비밀로 보호할 것

  • 스코어링 알고리즘의 구체적 가중치와 파라미터 (변동성 x 0.23 + 거래량 x 0.17 등)
  • AI 모델의 학습 데이터 및 하이퍼파라미터
  • 매치메이킹 밸런싱의 세부 로직
  • 유저 행동 분석 데이터 및 인사이트
  • 사업 전략, 파트너십 계획, 수익 모델 세부사항

영업비밀 보호 3요건 (모두 충족 필수)

1. 비공지성
공개되지 않은 정보일 것. NDA 하에 공유하는 것은 OK
2. 경제적 유용성
경쟁 우위를 제공하거나 독자 개발에 상당한 비용이 필요할 것
3. 비밀관리성
객관적으로 인식 가능한 비밀 관리 조치를 취할 것

구체적 실행 방법

  • 모든 알고리즘 문서에 "CONFIDENTIAL / 대외비" 표기
  • 접근 권한 제한 — 비밀번호 보호, Git 리포지토리 접근 제어
  • 외부 공유 시 반드시 NDA(비밀유지계약서) 체결 (멘토 포함)
  • 영업비밀 원본증명 서비스 (tradesecret.or.kr) 활용 — 타임스탬프 해시 생성으로 존재 시점 증명
  • 개발 로그에 상세 타임스탬프 기록 유지
Layer 5

비법적 해자 (Business Moat) — 법보다 강한 방어

법적 보호는 기본이지만, 진짜 방어벽은 실행 속도와 네트워크 효과입니다

솔직한 현실: 아이디어는 보호할 수 없다

  • "MLB 판타지리그를 주식에 적용한다"는 컨셉은 이미 해외에서 시도됨 (Fantasy Stock League, MoneyCloud 등). 이 컨셉 자체는 법적 독점이 불가능합니다.
  • 하지만 아이디어를 처음 생각한 사람이 아니라, 처음 제대로 실행한 사람이 이깁니다. Facebook은 최초의 SNS가 아니었고, iPhone은 최초의 스마트폰이 아니었습니다.
  • 컨셉을 보고 카피한다 해도, 스코어링 시스템 설계, UX 디테일, 데이터 파이프라인, 유저 커뮨니티는 카피할 수 없습니다. 이것들이 진짜 해자입니다.
⚡

실행 속도

누군가 컨셉을 보고 따라하려면 최소 6~12개월. 그 사이 MVP를 출시하고 유저를 확보하면 선점 효과 발생.

네트워크 효과

판타지리그는 본질적으로 소셜 게임. 친구가 있는 플랫폼에서 플레이. 유저가 유저를 데려오는 구조 = 전환 비용 급상승.

📊

데이터 해자

시즌별 유저 데이터, 리그 히스토리, 행동 패턴은 시간이 쌓일수록 가치 증가. 후발주자가 복제 불가능.

🤖

AI 스코어링 고도화

실사용 데이터로 학습되는 AI 모델. 운영할수록 정확도 증가. 코드를 복사해도 학습 데이터와 경험은 복사 불가.

🤝

파트너십
